# 2ITE WEB – Project Assignment

## Instructions

**Year Project Assignment:** Modern Web Presentation  
**Subject:** Web Technologies (2nd year)

### Goal
Create a comprehensive, fully optimized website on any topic using only core web technologies, and document the full development process.

## 1. Technical Specifications

- **Base stack:** Pure HTML5, CSS3 (modern layouts such as Flexbox/Grid), and Vanilla JavaScript (ES6+).
- **Restrictions:** JavaScript frameworks (React, Vue, Angular) and CSS frameworks (Bootstrap, Tailwind) are not allowed. The goal is to demonstrate low-level understanding.
- **Tools:** IDE is optional by student choice. AI tools may be used for generating content, code, or graphics.

## 2. Optimization Areas

The student must implement and document the following areas:

1. **Performance**
	- Minimize HTTP requests
	- Optimize image size and formats
	- Use lazy loading
	- Minify code

2. **SEO**
	- Proper semantic structure
	- Meta tags
	- Structured data
	- Valid HTML
	- `sitemap.xml`
	- `robots.txt`

3. **Accessibility**
	- Sufficient color contrast
	- ARIA attributes
	- Keyboard navigation
	- Screen reader support (WCAG standards)

4. **Social Media**
	- Open Graph implementation (Facebook, LinkedIn)
	- X (Twitter) Cards

5. **UI/UX**
	- Responsive design (Mobile First)
	- Intuitive navigation
	- Clear visual hierarchy
	- Typography best practices

6. **AI Integration**
	- Use AI during design/development (e.g., text generation, code analysis, or creation of unique graphic elements)

## 3. Documentation (`README.md`)

The `README.md` file in the project root must be detailed and include:

- **Introduction:** Project name, topic description, and link to live site (GitHub Pages)
- **Used technologies:** List of technologies and IDE version
- **Directory structure:** Description of project file organization
- **Technical analysis:** For each of the 6 optimization areas (section 2), include:
  - Theoretical explanation of the solution
  - Code snippet with explanation of how the issue is solved
- **AI Log:** List of useful prompts you used and what AI contributed to the project
- **Installation and run:** Instructions for local setup (e.g., Live Server)
- **Gallery:** Desktop and mobile screenshots and key feature previews

## 4. Submission Method

- The project must be uploaded to a public GitHub repository owned by the student.
- The website must be deployed using GitHub Pages.
- Submit only the link to the GitHub repository in the submission system.

## Grading Criteria

- **Functionality and validity:** Compliance with technical limits and clean code
- **Optimization quality:** Real impact of optimizations (e.g., measured with Google Lighthouse)
- **Documentation depth:** Ability to clearly explain your own code and process
- **Visual quality:** Aesthetics and usability of the final website
